# OpenCode Agent 工作流程规范

本文档定义了 OpenCode Agent 执行任务的规范和流程。

## 🎯 基本原则

1. **一次只执行一个任务** - 专注于当前任务，不发散
2. **测试驱动** - 每个功能必须有测试验证
3. **持续集成** - 每个任务完成后立即提交
4. **迭代改进** - 根据反馈不断优化

---

## 📋 工作流程

### 阶段 1: 任务分析

**执行步骤**:
1. 读取任务描述
2. 分析需求和技术方案
3. 检查现有代码结构
4. 确定实现策略

**输出**:
- 任务理解摘要
- 技术方案概要
- 可能遇到的问题预测

---

### 阶段 2: 代码实现

**执行步骤**:
1. 编写/修改代码
2. 遵循项目代码规范
3. 添加必要的注释
4. 确保代码可读性

**约束**:
- 不要修改与当前任务无关的代码
- 保持向后兼容性
- 使用项目已有的技术栈

---

### 阶段 3: 功能测试

**执行步骤**:
1. 运行单元测试（如果有）
2. 手动验证功能
3. 检查边界情况
4. 验证错误处理

**通过标准**:
- [ ] 功能按预期工作
- [ ] 无明显 bug
- [ ] 错误提示清晰

---

### 阶段 4: 文档更新

**执行步骤**:
1. 更新 README（如需要）
2. 添加/更新代码注释
3. 记录重要变更

---

### 阶段 5: Git 提交

**执行步骤**:
1. `git add -A`
2. `git commit -m "描述信息"`
3. `git push origin main`

**提交信息格式**:
```
类型: 简短描述

- 详细变更点 1
- 详细变更点 2
```

类型包括:
- `Feat`: 新功能
- `Fix`: 修复
- `Refactor`: 重构
- `Docs`: 文档
- `Test`: 测试

---

### 阶段 6: 任务更新

**执行步骤**:
1. 标记当前任务为完成
2. 根据执行结果创建新任务（如需要）
3. 更新任务优先级

---

## 🚫 禁止事项

- ❌ 一次修改多个不相关的功能
- ❌ 提交未经测试的代码
- ❌ 提交带有敏感信息的代码
- ❌ 破坏现有功能
- ❌ 过度工程化

---

## ✅ 检查清单

每个任务执行前确认:
- [ ] 理解任务需求
- [ ] 了解项目结构
- [ ] 备份重要数据

每个任务执行后确认:
- [ ] 功能测试通过
- [ ] 代码已提交
- [ ] 文档已更新
- [ ] 任务状态已更新

---

## 📝 示例工作流

```
任务: 添加用户登录功能

1. 分析
   - 需要用户认证
   - 使用 JWT token
   - 需要密码加密

2. 实现
   - 创建 auth.py
   - 实现 login() 函数
   - 实现密码验证

3. 测试
   - 测试正常登录
   - 测试错误密码
   - 测试 token 生成

4. 文档
   - 更新 API 文档
   - 添加使用示例

5. 提交
   - git add .
   - git commit -m "Feat: 添加用户登录功能"
   - git push

6. 更新任务
   - 标记任务完成
   - 创建新任务: "添加用户注册功能"
```

---

## 🔄 反馈循环

如果任务执行失败:
1. 分析失败原因
2. 修复问题
3. 重新测试
4. 再次提交
5. 记录教训

---

## 📊 质量指标

- 代码可读性
- 测试覆盖率
- 文档完整性
- 提交规范性
