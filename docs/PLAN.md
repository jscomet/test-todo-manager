# Todo Manager 项目计划 (Project Plan)

## 1. 项目目标

### 1.1 项目定位
**Todo Manager** 是一个简洁的命令行待办事项管理工具，面向终端用户。

**AI 角色**: AI 作为开发者，通过执行器自动开发和完善这个工具。

### 1.2 愿景
实现「定义开发任务 → AI 自动开发 → 测试验证 → 代码提交」的闭环，让 AI 持续改进 todo 管理器本身。

### 1.3 核心目标
- **功能完整**：覆盖任务全生命周期管理（增删改查、优先级、标签等）
- **代码质量**：模块化架构、高测试覆盖率
- **AI 驱动开发**：执行器驱动 AI 自动完成开发任务
- **用户友好**：清晰的文档、直观的 CLI 交互

### 1.4 重要区分

| 概念 | 说明 |
|------|------|
| **Todo 应用** | 给用户用的 CLI 工具，管理用户的待办事项 |
| **开发任务** | 本 PLAN 中定义的任务，是开发 todo 应用的工作项 |
| **AI 角色** | 开发者，不是 todo 应用的使用者 |
| **用户数据** | `data/tasks.json` 存储用户的真实待办，不是开发任务 |

## 5. 里程碑 3：BabyAGI 执行器 (M3)

### 5.1 目标
开发一个极简的 AI 驱动执行器，实现任务的自动执行与修复。

**核心设计**:
- **Agent 极简**: Prompt 模板 + OpenCode 调用
- **记忆简化**: 文件-based，**无向量数据库**
- **工具零实现**: **完全依赖 OpenCode 内置工具**
